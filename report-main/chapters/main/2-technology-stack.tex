\chapter{Nền tảng công nghệ}
\textit{Chương này phân tích lý do lựa chọn các công nghệ cốt lõi cho nền tảng ôn luyện tiếng Anh tích hợp AI. Mỗi quyết định được đánh giá dựa trên các yêu cầu đặc thù của dự án: xử lý nội dung giáo dục đa phương tiện, tích hợp AI inference, khả năng mở rộng cho lượng người dùng lớn, và tốc độ phát triển phù hợp với tiến độ đồ án.}

\section{Phát triển frontend}

\subsection{Phân tích và lựa chọn frontend framework}

Nền tảng ôn luyện tiếng Anh có ba yêu cầu frontend đặc thù: (1) SEO tốt để người dùng tìm thấy nội dung luyện thi qua search engine, (2) tối ưu hiệu năng tải trang cho nội dung giàu hình ảnh và audio, và (3) hỗ trợ cả trang tĩnh (giới thiệu, blog) lẫn trang động (dashboard, làm bài thi). Ba framework được đánh giá dựa trên các tiêu chí này:

\textbf{AngularJS} (Google) sử dụng kiến trúc MVC với two-way data binding, nhưng không hỗ trợ server-side rendering mặc định --- hạn chế nghiêm trọng đối với yêu cầu SEO của nền tảng giáo dục. Bundle size lớn do tính chất monolithic cũng ảnh hưởng đến thời gian tải ban đầu. \cite{angularjs}

\textbf{ReactJS} (Meta) mạnh về xây dựng UI tương tác nhờ Virtual DOM và kiến trúc component-based, nhưng client-side rendering mặc định gây bất lợi cho SEO. Cần bổ sung nhiều library riêng cho routing, state management và image optimization. \cite{angular_react}

\textbf{Next.js} xây dựng trên React, bổ sung hybrid rendering (SSR + SSG), automatic code splitting và built-in image optimization --- giải quyết trực tiếp cả ba yêu cầu đặc thù của dự án mà không cần cấu hình thêm. \cite{next_react_angular}

Nhóm chọn \textbf{Next.js} vì framework này đáp ứng trực tiếp các yêu cầu đặc thù của nền tảng giáo dục:

\begin{enumerate}
\item \textbf{SEO cho nội dung luyện thi}: Nền tảng phụ thuộc vào organic search traffic để tiếp cận học viên. SSR tích hợp sẵn bảo đảm nội dung bài thi, bài viết hướng dẫn được search engine index đầy đủ --- yếu tố quyết định khả năng tăng trưởng người dùng mà không cần ngân sách quảng cáo lớn.

\item \textbf{Tối ưu tài liệu giàu hình ảnh}: Tài liệu luyện thi chứa nhiều hình ảnh minh họa, sơ đồ và biểu đồ. Next.js Image tự động phân phối hình ảnh đúng kích thước ở định dạng WebP, giảm đáng kể thời gian tải --- đặc biệt quan trọng cho người dùng ở vùng có kết nối Internet chậm.

\item \textbf{Hybrid rendering phù hợp kiến trúc}: Trang giới thiệu và blog dùng SSG (tải nhanh, cache tốt), trong khi dashboard làm bài và kết quả AI dùng SSR (dữ liệu luôn cập nhật). Next.js cho phép kết hợp cả hai trong cùng một ứng dụng mà không cần tách dự án.

\item \textbf{TypeScript xuyên suốt}: Sử dụng chung TypeScript với backend NestJS, giúp nhóm tái sử dụng type definition (DTO, API response) giữa frontend và backend, giảm lỗi runtime và tăng tốc phát triển.
\end{enumerate}

\section{Phát triển máy chủ}
\subsection{Phân tích và lựa chọn backend framework}

Hệ thống có hai nhóm workload rõ rệt: (1) application logic hướng người dùng (authentication, CRUD, real-time notification) và (2) AI inference (chấm bài, phân tích giọng nói, sinh đề). Sự phân tách này dẫn đến kiến trúc dual-backend, đòi hỏi đánh giá framework theo hai vai trò khác nhau.

\textbf{FastAPI} (Python) --- framework API hiệu năng cao với async native và tự động sinh OpenAPI documentation. Lợi thế cốt lõi: Python là ngôn ngữ chủ đạo của AI/ML ecosystem. Các library thiết yếu cho dự án (Whisper, librosa, LangChain, PyTorch) chỉ hỗ trợ Python hoặc ưu tiên Python. Dùng framework non-Python cho AI backend sẽ tạo overhead xuyên ngôn ngữ không cần thiết. \cite{fastapi}

\textbf{NestJS} (TypeScript/Node.js) --- framework modular với dependency injection, lấy cảm hứng từ Angular. Mô hình event-driven, non-blocking I/O của Node.js phù hợp cho workload concurrency cao (WebSocket, real-time dashboard). Chia sẻ TypeScript với frontend Next.js giúp tái sử dụng type definition và giảm context switching cho nhóm phát triển. \cite{nestjs}

\textbf{ASP.NET Core} (C\#) và \textbf{Spring Boot} (Java) --- đều là framework enterprise-grade với performance và reliability cao. Tuy nhiên, cả hai đều có chu kỳ phát triển dài hơn, không phù hợp với tiến độ ngắn của đồ án. Quan trọng hơn, AI ecosystem của C\# và Java hạn chế hơn nhiều so với Python --- việc tích hợp AI model cuối cùng vẫn đòi hỏi service Python riêng, làm tăng độ phức tạp kiến trúc mà không mang lại lợi ích bù đắp. \cite{aspnet, springboot}

\textbf{Quyết định}: Kiến trúc dual-backend kết hợp NestJS và FastAPI, mỗi framework đảm nhận đúng vai trò mà nó phát huy hiệu quả nhất:

\begin{itemize}
\item \textbf{NestJS --- Application backend}: Xử lý toàn bộ tương tác người dùng, authentication, session management, CRUD và real-time notification. Kiến trúc modular của NestJS giúp tổ chức hệ thống thành các module độc lập (user, exam, payment), dễ phân chia công việc trong nhóm 4 người. TypeScript chung với Next.js frontend cho phép tái sử dụng DTO và API response type, giảm lỗi tích hợp.

\item \textbf{FastAPI --- AI backend}: Chuyên trách các tác vụ AI inference: speech-to-text (Whisper), phân tích phát âm (librosa, Parselmouth), chấm điểm bài luận và tự động hóa với LangChain. FastAPI được chọn vì đây là lựa chọn duy nhất cho phép truy cập trực tiếp vào toàn bộ AI/ML ecosystem của Python mà không cần lớp cầu nối. Async native của FastAPI cũng phù hợp với các inference task kéo dài (5--30 giây cho speech analysis).
\end{itemize}

\subsection{Phân tích và lựa chọn database}

Hệ thống lưu trữ hai nhóm dữ liệu có tính chất khác nhau: (1) dữ liệu transactional có cấu trúc cố định (user, payment, permission) đòi hỏi ACID compliance nghiêm ngặt, và (2) dữ liệu nội dung giáo dục với schema thay đổi thường xuyên (câu hỏi thi, learning analytics, AI-generated content). Sự phân tách này dẫn đến kiến trúc dual-database.

\textbf{Quyết định}: Kết hợp PostgreSQL và MongoDB thành kiến trúc dual-database bổ trợ:

\textbf{PostgreSQL} cho web application lõi (NestJS backend):
\begin{itemize}
\item Các workflow transactional quan trọng (authentication, ghi danh, thanh toán, quản trị) đòi hỏi ACID compliance nghiêm ngặt mà PostgreSQL bảo đảm đáng tin cậy.
\item Relational model phù hợp cho dữ liệu có quan hệ rõ ràng: user $\rightarrow$ enrollment $\rightarrow$ exam result $\rightarrow$ payment.
\item Hỗ trợ JSONB cho phép lưu metadata linh hoạt khi cần, không bị giới hạn hoàn toàn bởi schema cứng. \cite{postgresql}
\end{itemize}

\textbf{MongoDB} cho nội dung giáo dục và AI analytics (FastAPI backend):
\begin{itemize}
\item Cấu trúc câu hỏi thi thay đổi thường xuyên khi bổ sung định dạng mới (IELTS, TOEIC, custom test). Document model cho phép iteration nhanh mà không cần database migration.
\item Kết quả phân tích AI (pronunciation score, essay feedback, learning path) có cấu trúc phức tạp và lồng nhau, ánh xạ tự nhiên sang JSON document.
\item Aggregation pipeline mạnh cho tính toán xu hướng hiệu suất học tập qua nhiều lần làm bài. \cite{mongodb}
\end{itemize}

\textbf{AWS} được chọn vì ba lý do trực tiếp liên quan đến dự án:
\begin{itemize}
\item \textbf{MongoDB Atlas native integration}: Database MongoDB của hệ thống chạy trên MongoDB Atlas --- dịch vụ này tích hợp native với AWS, giúp giảm network latency và đơn giản hóa backup/monitoring so với việc chạy trên cloud provider khác.
\item \textbf{AI service dự phòng}: AWS Transcribe và Polly cung cấp speech processing fallback khi Whisper self-hosted gặp sự cố hoặc quá tải, bảo đảm tính sẵn sàng cho tính năng luyện nói.
\item \textbf{Danh mục dịch vụ đầy đủ}: Từ compute (EC2, Lambda) đến storage (S3) và CDN (CloudFront), AWS cung cấp toàn bộ infrastructure cần thiết trong cùng một ecosystem, giảm nhu cầu tích hợp bên thứ ba. \cite{aws}
\end{itemize}
